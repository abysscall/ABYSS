# ADR-0005 — State as an Independent Module

**Status:** Proposed
**Date:** 2026

---

## Context

-e Currently balances and nonces are stored directly inside Chain in abyss-core. This is appropriate for the devnet phase, where the only state is account balances and nonces. However, mainnet ABYSS will include validators, smart contracts, staking, governance, treasury logic, and social-layer anchors. Keeping all of this inside Chain will produce excessive coupling and make subsystems impossible to test or evolve independently.

---

## Decision

-e Introduce a dedicated State module, initially as a refactor of abyss-core.

Target structure:

 Chain
 └── State
 ├── Accounts (balances, nonces)
 ├── Validators (staking, slashing)
 ├── Treasury
 ├── Contracts (EVM-compatible + ZK circuit)
 ├── Governance
 └── Storage (social-layer anchors)

Chain owns consensus and block production. State owns all persistent data.

---

## Consequences

-e Advantages:
- Modular architecture — each subsystem can be tested and evolved independently
- Lower coupling — adding Contracts does not require touching Validator logic
- Easier to implement persistent storage — the State boundary is the persistence boundary

This ADR is Proposed, not Accepted, because the refactor has not yet begun.
It is documented now so that new code does not increase the coupling it will need to break.
