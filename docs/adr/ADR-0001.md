# ADR-0001 — Domain Types Instead of Primitive Types

**Status:** Accepted
**Date:** 2026

---

## Context

-e Blockchain software manipulates critical entities: addresses, coins, hashes, transaction and block identifiers. Representing these as primitive types (String, u64, Vec<u8>) increases the probability of accidental misuse — passing a block hash where a transaction hash is expected, for example — and weakens compile-time safety. The compiler cannot catch semantic errors when all values share the same underlying type.

---

## Decision

-e ABYSS represents every important blockchain concept as a dedicated Rust type.

Examples in abyss-core:
- Address — a validated account identifier
- Coin — an amount of AC with overflow-safe arithmetic
- TransactionId — a typed wrapper around a transaction hash
- BlockHash — distinct from Hash256 at the type level
- Hash256 — raw 32-byte hash output

Primitive aliases (type Address = String) are intentionally avoided because they provide no compiler enforcement.

---

## Consequences

-e Advantages:
- Stronger type safety — the compiler rejects whole classes of mistakes
- Clearer APIs — function signatures document their intent
- Better compiler diagnostics — errors are caught at call sites

Trade-offs:
- More boilerplate for conversions between types
- New contributors must learn the type vocabulary before reading code
