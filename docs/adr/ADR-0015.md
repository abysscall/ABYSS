# ADR-0015 — Native-First Launch; ERC-20 Is Not the Product

**Status:** Accepted
**Date:** 2026
**Supersedes:** the "ERC-20 bridge period" clause of ADR-0012

---

## Context

Earlier planning (ADR-0012) treated an ERC-20 token on Ethereum as the primary
vehicle for AC during the fundraising period, with a 1:1 migration to native AC
at mainnet genesis. Under that framing the ERC-20 was on the critical path and
the independent network was, in practice, a later milestone.

This inverts the priority ABYSS is built on. Per ADR-0000 ("The architecture is
the product"), the product is the independent ABYSS network and its native coin
AC — not a token contract on someone else's chain. Anchoring the project to
Ethereum's execution and economics contradicts the privacy, decentralisation,
and self-sovereignty pillars (ADR-0010).

## Decision

**1. The independent ABYSS network and native AC are the primary goal.**
Every new architectural decision must move the project toward launching and
operating an independent chain: consensus, storage, node RPC, shielded
execution, and native economics. Work is prioritised by how directly it advances
the native network.

**2. ERC-20 is no longer the primary path.**
AC is not defined as an Ethereum token. There is no ERC-20-first issuance on the
critical path and no "ERC-20 → native migration" as the central launch event.

**3. Interoperability comes later, via bridges and wrapped assets.**
If cross-ecosystem compatibility is needed, it is added *around* the native
network — bridge contracts and wrapped representations (e.g. wrapped AC on other
chains, wrapped external assets on ABYSS) — never by replacing native economics
with a foreign token standard. Interop is an optional edge, not the core.

**4. Fundraising is decoupled from token issuance.**
Before genesis there is no token of any kind — no ERC-20, no testnet allocation,
no escrow instrument as the primary mechanism. There is only a **Contribution
Registry**: a contributor pays in an accepted external currency (BTC, USDT,
USDC, bank transfer; Lightning later), and a signed **Contribution Receipt** is
recorded. Receipts accumulate into the **Genesis Allocation Registry**, and the
**Genesis Block itself mints the first AC** directly from those allocations. The
registry design is specified in ADR-0016.

## Consequences

- Engineering roadmap reorders toward the native network: multi-validator BFT
  consensus, persistent storage, JSON-RPC, and the ZK/shielded layer move ahead
  of any token-contract work.
- ADR-0012 stands for supply, split, sale stages, and vesting; only its ERC-20
  bridge/migration clause is superseded by this ADR.
- Investor-facing materials that describe AC as "ERC-20 → Native" must be
  reworded to "native AC, interop later via bridges/wrapped assets".
- ABYSS avoids inheriting Ethereum's fees, congestion, and transparency defaults
  for its base economy.

## Notes

The pre-genesis instrument is the Contribution Registry (ADR-0016): no temporary
token, no testnet allocation, no ERC-20. This ADR fixes the *direction*; a legal
review of accepting funds is still required before any contributions are taken.
