# ADR-0003 — Separate Block Header From Block Body

**Status:** Accepted
**Date:** 2026

---

## Context

-e Future versions of ABYSS will support light clients, efficient block synchronisation, Merkle proofs, and compact peer-to-peer messaging. All of these use cases require independent access to block metadata (height, timestamp, previous hash, Merkle root) without requiring the full transaction list to be transferred or validated.

---

## Decision

-e BlockHeader and Block are separate structures in abyss-core.

- The block hash is computed exclusively from the header.
- The transaction list belongs to the block body, not the header.
- Header verification can proceed without the body.
- The Merkle root in the header commits to the body's contents.

---

## Consequences

-e This architecture enables future support for:
- SPV (Simplified Payment Verification) light clients
- Merkle inclusion proofs for individual transactions
- Compact block relay (send header, request only missing transactions)
- Efficient peer-to-peer block announcement

There is no meaningful trade-off: separating header and body is strictly more capable than merging them.
